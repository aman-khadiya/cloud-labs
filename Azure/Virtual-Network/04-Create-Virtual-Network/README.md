# Day 04 - Create Virtual Network (VNet)

## 📌 Task

Create an Azure Virtual Network (VNet) named **devops-vnet** in the **West US** region with any valid IPv4 CIDR block.

---

## 🛠️ Services Used

- Azure Virtual Network (VNet)
- Azure Portal

---

## ⚙️ Configuration

| Setting | Value |
|---------|-------|
| Virtual Network Name | devops-vnet |
| Region | West US |
| Address Space | 10.0.0.0/16 |
| Subnet | default (10.0.0.0/24) |

---

## 🚀 Steps Performed

1. Logged into Azure Portal.
2. Opened **Virtual Networks**.
3. Clicked **Create**.
4. Entered:
   - VNet Name: `devops-vnet`
   - Region: `West US`
5. Configured IPv4 Address Space:
   - `10.0.0.0/16`
6. Kept the default subnet.
7. Reviewed the configuration.
8. Created the Virtual Network successfully.
9. Verified the VNet from the Azure Portal.
